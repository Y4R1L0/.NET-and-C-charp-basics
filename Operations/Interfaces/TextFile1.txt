Интерфейсы

В ООП интерфейсы - разновидность абстрактного класса у которого:
все элементы public 
все абстрактные 
только методы (и свойства)

Други значения этого термина:

правило взаимоодействия (подключения) - USB интерфейс
Набор для взааимодействия с пользователем - UI (user interface)
поставщики данных, Web Interface
"Видимая" часть модуля библиотеки или пространства имен

Зачем нужны интерфейсы (ООП)?


Абстракция синоним "обобщения" действие обратное наследованию,"собирательный термин", расширению
Абстранирование создание родительских классов объединяющих конкретные сущности

Интерфейсы считаются высшей формой абстракции - объединяют сущности по минимальному количеству признаков (обычно 1)
в т.ч. и вообще без признаков - интерфейсы маркеры

С обратной стороны появляется возможность резделить сущности на группы:
которые реализуют / которые не реализуют даанный интерфейс

Причем одна сущност может быть  участником нескольких групп, реализуя разные интрефейсы

Реализация / наследование (расширение)
наследование интерфейса - создание нового интрефейса, расширяющий данный 
(не создающего реализацию остается абстрактным)
реализация интерфейса - создание класса, реализующего методы интерфейса
(абстракция снимается, можно создавать обьекты)


Языковые конструкции (С#)

p is IPrintable - true если обьект "p" реализует IPrintable, иначе false
проверка наличия интерфейса / или наследования

p as IPrintable - ссылочное преобразование типа
